# IdeaVault Flow

IdeaVault Flow — это единый ассистент для сбора и работы с идеями через Telegram и Web UI.

Позволяет быстро сохранять мысли, искать по ним, анализировать и задавать вопросы к базе знаний (RAG) без сложной инфраструктуры.

**Зачем:** связка **захват → хранение → поиск → обзор и ответы** без тяжёлой инфраструктуры; RAG по файлам в `data/knowledge` и публичным **GitHub raw**.

## Что умеет система

### 📥 Захват

- **Telegram capture** — текст, голос (STT при `OPENAI_API_KEY`), фото как заметка-артефакт без vision в чате; заметка `+`, свободный текст в зависимости от режима.
- **Мультимодальность** — `POST /api/capture` (картинка + подпись + проект); опционально LLM/vision при включённом слое OpenAI.

### 🧠 Работа с данными

- **FTS-поиск** по заметкам (нормализация запроса, AND → OR → одиночные токены).
- **`GET /api/review`** и команды **`/review` / `/next` в боте** — снимок области и эвристические следующие шаги.

### 📚 RAG

- **RAG (MVP)** — локальные `.md` / `.txt` и файлы с GitHub (whitelist путей, raw); индексация и ответы с указанием источников.

### 🌐 Интерфейсы

- **Web UI** — дашборд проектов, чат (vault / RAG), список заметок, при RAG — привязка GitHub и переиндексация.

## 📌 Status

**MVP** → готов к демонстрации и дальнейшему развитию.

Подходит для:

- сбора идей и заметок по проектам;
- RAG по локальным файлам и GitHub;
- сценариев **Telegram-first** с дублированием в веб-интерфейсе.

## 🚀 Quick Demo (5 минут)

1. Запустите API: `uvicorn app.main:app --reload --host 0.0.0.0 --port 8000`.
2. Откройте **[http://127.0.0.1:8000/ui](http://127.0.0.1:8000/ui)**.
3. Создайте проект на дашборде и откройте его карточку.
4. Добавьте заметку: **`POST /api/items`** с полем `project`, бот **`+ текст`** (после `/set <имя>`) или формы Web UI по ссылкам со страницы проекта.
5. Положите файл (например `.md`) в **`data/knowledge/<имя_проекта>/`** (имя совпадает с проектом) или используйте пример **`data/knowledge/_global/example.md`**.
6. На карточке проекта нажмите **«Переиндексировать RAG»** (тот же контур, что `POST /api/rag/index`).
7. В боте или веб-чате переключите режим в **RAG** (`/mode rag` в Telegram или кнопка на `/ui/project/{name}`) и задайте вопрос по содержимому базы.

Опционально: **`/docs`**, **`GET /api/stats`** (поля `rag_*`), сценарий **`/mode vault`** для вопросов только по заметкам. Роль ассистента и сценарии — в **`IDEA.md`**.

**Результат:** вы получите ответ на основе ваших файлов и заметок с указанием источников.

## 🧩 Как это работает (в 1 минуту)

1. Вы сохраняете идеи (Telegram или UI)
2. Они попадают в SQLite + индексируются (FTS)
3. При RAG добавляются документы (локально / GitHub)
4. Система ищет релевантный контекст
5. LLM (опционально) формирует ответ только на основе найденных данных

## 🌐 Web UI

Web UI — основной способ работы с проектами:

- создаёте проект
- добавляете заметки
- подключаете RAG
- общаетесь в чате

**Основные страницы**

| Путь | Назначение |
|------|------------|
| `/ui` | Дашборд: проекты, показатели, переход в карточку |
| `/ui/project/{name}` | Проект: чат (vault / RAG), переиндексация RAG, локальные файлы знаний, ссылки на заметки и RAG |
| `/ui/items` | Список заметок, фильтр по проекту, перенос между проектами |
| `/ui/rag` | Привязка GitHub (`?project=…`), валидация путей через raw.githubusercontent.com |

**Типовой поток:** дашборд → карточка проекта (чат и RAG) → при необходимости заметки (`/ui/items`) или настройка GitHub (`/ui/rag`). Без общего верхнего меню; навигация с дашборда и со страницы проекта. Прямые URL `/ui/rag`, `/ui/items`, **`/docs`** доступны. Корневой `GET /` в JSON указывает `"ui": "/ui"`. Для совместимости сохранён `POST /ui/projects/set`.

Стек: Jinja2, только формы; при удалении проекта (не служебного **Null**) заметки переносятся в **Null** с префиксом в тексте.

## Запуск Telegram-бота

```powershell
python -m app.bot.main
```

Нужен непустой `TELEGRAM_BOT_TOKEN` в `.env`. Long polling, без webhook.

**Проект в боте:** задайте имя **`/set <имя>`**; переключение между уже существующими проектами — **`/project`** (inline-кнопки; при смене проекта контекст чата сбрасывается). Глобальная область — **`/clear`**.

Telegram — быстрый способ захвата и работы с идеями без открытия UI.

### Команды (основные)

| Команда | Описание |
|---------|----------|
| `/start` | Краткая справка |
| `/set` | Текущий проект: `/set <имя>` или просмотр контекста (вместе с `/current`) |
| `/project` | Выбор из проектов, уже есть в БД (кнопки); новый проект только через `/set <имя>` |
| `/clear` | Сброс области на глобальную (без проекта) |
| `/review` | Снимок текущей области (как обзор заметок) |
| `/next` | Эвристические следующие шаги |
| `/mode` | Режим ответов на свободный текст: `vault` (заметки) или `rag` (база знаний) |
| `/index` | Переиндексация RAG для текущего проекта (локальные файлы + GitHub по привязке) |

### Расширенные команды и ввод

| Команда / ввод | Описание |
|----------------|----------|
| `/current`, `/projects` | Текущий проект и список имён из заметок |
| `/resetchat` | Новый поток чата для пары (проект, режим) без удаления строк в БД |
| `/rag_bind`, `/rag_paths` | Привязка `owner/repo`, whitelist путей для raw |
| `/stats` | Сводка, в т.ч. счётчики RAG |
| `+ текст` | Создать заметку (дедуп по нормализованному тексту в рамках проекта) |
| Обычный текст | В `vault` — ответ по заметкам; в `rag` — вопрос по чанкам знаний |
| Голосовое сообщение | Файл в `data/voice/…`, STT (Whisper-совместимый API), далее как текст |
| Фото | Файл сохраняется, создаётся заметка-артефакт; описание — вручную в тексте заметки |

История чата привязана к потоку `chat_threads` с ключом *(user, project, mode)*.

## 📚 RAG (MVP)

- **Локально** — каталоги `data/knowledge/<проект>/` и **`data/knowledge/_global/`** (`.md`, `.txt`).
- **GitHub** — только публичные **raw**-URL; привязка репозитория и путей в боте или на `/ui/rag`.
- **Индекс** — команда **`/index`** в боте, кнопка на карточке проекта или **`POST /api/rag/index`**.
- **Режим ответов** — **`/mode rag`** (бот) или переключатель на **`/ui/project/{name}`**.

При выбранном проекте в RAG учитываются чанки проекта и глобальные (`project IS NULL`); без проекта — только глобальные. К ответу могут добавляться релевантные **заметки** из vault.

Таблицы: `rag_documents`, `rag_chunks`, FTS `rag_chunks_fts`. Детали пайплайна — в **`app/rag/`** (indexer, retriever, источники).

## Архитектура (кратко)

1. **Handlers** (Telegram) и **роуты** (FastAPI) — тонкий ввод/вывод.
2. **Services** — сценарии поиска, обзора, захвата, LLM, RAG-ответов.
3. **Repositories** — SQLite через SQLAlchemy.
4. **`app/rag/`** — chunking, локальные файлы + GitHub raw, indexer, retriever; точка замены FTS на **ChromaDB** — `retriever`.
5. **Integrations** — OpenAI (текст, при необходимости vision/STT).
6. Схема: `items`, RAG-таблицы, `project_registry` (в т.ч. системный **Null**), `sessions` (`chat_mode`, …); FTS **`items_fts`**, **`rag_chunks_fts`**.

## Endpoints (JSON API, префикс `/api`)

| Метод | Путь | Назначение |
|--------|------|------------|
| GET | `/api/health` | Liveness: `{"status":"ok"}` |
| GET | `/api/stats` | Сводка vault + RAG: `items_*`, `rag_documents_total`, `rag_chunks_total`, `rag_projects_with_docs` |
| GET | `/api/items` | Список заметок (`project`, `limit`) |
| POST | `/api/items` | Создать текстовую заметку (JSON) |
| GET | `/api/search` | `?q=` → `{"query","items"}` |
| GET | `/api/projects` | Уникальные проекты |
| POST | `/api/projects/current`, … | Текущий проект для `user_id` |
| GET | `/api/review` | Снимок как в боте (`?project=` опционально) |
| POST | `/api/review/ask`, … | Вопрос + retrieval → `answer`, `sources`, `next_steps` |
| POST | `/api/capture` | Multipart: `file` (image), опционально `caption`, `project` |
| POST | `/api/rag/index` | JSON `{"project": "имя" \| null}` — переиндексация `data/knowledge/…` и GitHub |

### RAG: файлы и GitHub (детали)

- **Локально:** `.md` / `.txt` в `data/knowledge/<имя_проекта>/` (совпадает с `/set`) или **`data/knowledge/_global/`**.
- **GitHub:** `/set course` → `/rag_bind owner/repo main` → опционально `/rag_paths README.md docs/lecture.md` → **`/index`** или `POST /api/rag/index` с телом `{"project":"course"}`. Без OAuth.

### Мультимодальный пример (`curl.exe`)

```powershell
curl.exe -s -X POST "http://127.0.0.1:8000/api/capture" `
  -F "file=@C:\path\to\photo.png" `
  -F "caption=Идея интерфейса" `
  -F "project=demo-course"
```

Ответ: `project`, `caption`, `capture`, `item_id`, `status: "saved"`. Без LLM — текстовый fallback; с ключом и успехом API — краткое описание изображения.

## Поиск (FTS) и ограничения

Запросы нормализуются без тяжёлого NLP; FTS5: AND по токенам, при пустом результате — OR и объединение одиночных токенов. При обзорных формулировках и известном проекте возможен fallback списком последних заметок. **Нет** стемминга и эмбеддингов в MVP.

## Слой LLM и логи

Retrieval **не заменяется** моделью: сначала контекст из БД, затем короткая генерация **только** по переданным фрагментам. Промпты ужаты под Telegram. Логи `llm_*` в консоли; `LLM_DEBUG_LOGGING` — дополнительные безопасные поля.
